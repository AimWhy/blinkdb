# Overview

> No matter if you're using [Redux](https://redux.js.org/), [MobX](https://mobx.js.org/README.html), or just a simple JS `Map()` -
> The data structures they use are not optimized for large-scale data, often buckling under the weight of having to handle
> a couple thousand objects at once. So, **why not bring the queries, indices, convenience and raw speed of databases to the frontend?**

**blinkDB** is a highly optimized, production ready database just for your frontend. It uses the same
techniques & data structures as existing databases in order to speed up the retrieval of items,
resulting in blazing fast performance, regardless if you're working with **10.000** or **1.000.000** database entities.
